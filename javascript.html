JavaScript: Powering Interactivity

Discover how JavaScript brings dynamic behaviour and interactivity to web applications.

Theoretical Concepts of JavaScript

What is JavaScript?

JavaScript is a lightweight, interpreted programming language that runs directly in the browser. It is the only programming language that browsers understand natively, which makes it an essential skill for any web developer. JavaScript allows you to make web pages dynamic and interactive — responding to user actions, updating content without reloading, validating form data, and much more. It was originally created for client-side scripting but with the introduction of Node.js it can also run on the server side, making it a full stack language.

DOM Manipulation

The Document Object Model (DOM) is the browser's representation of the HTML page as a tree of objects. JavaScript can access and modify this tree to dynamically change the content, structure, or styling of the page without reloading it. Common methods for selecting elements include document.getElementById(), document.querySelector(), and document.querySelectorAll(). Once you have a reference to an element, you can change its text (innerText), HTML content (innerHTML), CSS styles (style), or attributes.

Event Handling

Events are actions or occurrences that happen in the browser — things like a user clicking a button, moving the mouse, pressing a key, or submitting a form. JavaScript listens for these events using addEventListener(), which takes the event type (e.g., 'click', 'submit', 'keypress') and a function to run when the event happens. This event-driven model is fundamental to how interactive web pages work.

Variables, Functions and Conditionals

JavaScript supports all the core programming concepts. Variables are declared with let (for values that will change) or const (for values that will not change). Functions are reusable blocks of code defined with the function keyword or as arrow functions (() => {}). Conditionals (if, else if, else) allow the code to make decisions based on conditions. These building blocks form the foundation of any JavaScript programme.

Asynchronous Programming

JavaScript is single-threaded, but it handles tasks that take time (like fetching data from a server) asynchronously so the rest of the page does not freeze. This is done using callbacks, Promises, or the modern async/await syntax. The fetch() API is the standard way to make HTTP requests from JavaScript to retrieve or send data to a server without reloading the page. This is the basis of how modern web applications load data dynamically.

JavaScript and Cyber Security

From a Cyber Security perspective, JavaScript is important to understand because it is at the centre of many web vulnerabilities. Cross-Site Scripting (XSS) attacks involve injecting malicious JavaScript into a webpage to steal data or hijack sessions. Understanding how JavaScript runs in the browser, how it accesses the DOM, and how it communicates with servers helps in both identifying and preventing these kinds of attacks.

Practical Examples of JavaScript

Dynamic Content Update

Changing the text content of an element dynamically when a button is clicked, using innerText.

// Select the element with id="dynamicText"
// Then change its text content when the button is clicked

function changeText() {
  document.getElementById('dynamicText').innerText =
    'Text has been updated with JavaScript!';
}

Event Listener – Toggle Colour

Using addEventListener to toggle a box's background colour between two values when clicked.

// addEventListener listens for a 'click' event on the button
// When clicked, it reads the current colour of the box and switches it

document.getElementById('toggleColor').addEventListener('click', () => {
  const box = document.getElementById('colorBox');

  if (box.style.backgroundColor === 'rgb(59, 130, 246)') {
    box.style.backgroundColor = '#1e3a8a';  // dark blue
  } else {
    box.style.backgroundColor = '#3b82f6';  // lighter blue
  }
});

Form Validation

Using JavaScript to validate form input before it is submitted. This prevents empty or invalid data from being sent to the server. 

// Listen for the form's submit event
// e.preventDefault() stops the page from reloading on submit

document.getElementById('sampleForm').addEventListener('submit', (e) => {
  e.preventDefault();  // stop the default form submission behaviour

  const username = document.getElementById('username').value;
  const message  = document.getElementById('formMessage');

  if (username.trim() === '') {
    message.style.color = 'red';
    message.innerText = 'Please enter a username before submitting.';
  } else {
    message.style.color = 'green';
    message.innerText = 'Username "' + username + '" submitted successfully!';
  }
});

Dynamic List Builder

Creating new HTML elements dynamically and adding them to the page using createElement() and appendChild(). 

// When the Add Item button is clicked:
// 1. Read the value from the input field
// 2. Create a new <li> element
// 3. Set its text to the input value
// 4. Append it to the list
// 5. Clear the input field

document.getElementById('addItem').addEventListener('click', () => {
  const input = document.getElementById('listInput');
  const list  = document.getElementById('dynamicList');

  if (input.value.trim() !== '') {
    const li = document.createElement('li');   // create a new list item
    li.innerText = input.value;                // set the text
    list.appendChild(li);                      // add it to the list in the DOM
    input.value = '';                          // clear the input
  }
});
