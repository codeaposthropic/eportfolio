Theoretical Concepts of Full Stack Development

What is Full Stack Development?

Full Stack Development refers to the ability to work on both the front-end (client-side) and the back-end (server-side) of a web application, covering the complete development process from the user interface all the way through to the database. A full stack developer understands how every layer of a web application works and how they interact with each other. The word "stack" refers to the collection of technologies used across all these layers. Rather than specialising in just one area, full stack developers can build and maintain entire applications independently.

Front-End Development

The front end is everything the user sees and interacts with directly in their browser. It is built using three core technologies: HTML for structuring content, CSS for applying visual styles and layout, and JavaScript for adding interactive behaviour. Front-end developers also often use frameworks and libraries to speed up development. Bootstrap is a popular CSS framework that provides ready-made responsive components. JavaScript frameworks like React, Vue.js, and Angular are widely used to build complex, dynamic user interfaces. The front end communicates with the back end by sending HTTP requests and receiving data (usually in JSON format) in return.

Back-End Development

The back end is the server-side part of an application — the code that runs on a web server rather than in the user's browser. It handles the business logic of the application, processes incoming requests from the front end, communicates with the database, and sends responses back. Common back-end programming languages and environments include Node.js (JavaScript on the server), Python with frameworks like Django or Flask, PHP, Ruby on Rails, and Java with Spring. The back end is responsible for things like user authentication, data processing, security checks, and serving the correct data to the correct user.

Databases

A database is where the application's data is permanently stored. When a user registers on a website, their information is saved to a database. When they log in again, the back end retrieves that data and uses it to authenticate them. Databases come in two main types: relational databases, which store data in structured tables with rows and columns (examples include MySQL, PostgreSQL, and SQLite), and non-relational (NoSQL) databases, which store data in a more flexible format such as documents or key-value pairs (examples include MongoDB and Redis). The back-end code communicates with the database using query languages like SQL.

Role of APIs

APIs (Application Programming Interfaces) are the mechanism that allows the front end and back end to communicate with each other. When you load a webpage and it fetches live data — like a weather forecast or a list of products — it is using an API call. The most common type in web development is a REST API, where the front end sends HTTP requests (GET, POST, PUT, DELETE) to specific URL endpoints on the server, and the server responds with data in JSON format. APIs also allow different applications to share functionality — for example, embedding a Google Map in a website uses Google's Maps API.

Version Control and Deployment

Version control is the practice of tracking and managing changes to code over time. Git is the most widely used version control system. Developers use Git to save snapshots of their code (commits), work on separate features without affecting the main codebase (branches), and collaborate with others (merging and pull requests). GitHub and GitLab are popular platforms for hosting Git repositories. Deployment refers to the process of making the application available on the internet. Platforms like GitHub Pages (used for static sites), Heroku, Netlify, Vercel, and AWS are commonly used to host and deploy web applications.

DevOps and Collaboration

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops) with the aim of shortening development cycles and delivering updates more reliably. Key DevOps practices include Continuous Integration (CI), where code changes are automatically tested whenever they are pushed to the repository, and Continuous Delivery/Deployment (CD), where tested code is automatically deployed to production. Tools like GitHub Actions, Jenkins, and Docker are commonly used in DevOps workflows. For full stack developers, understanding basic DevOps practices helps in deploying applications efficiently and collaborating with operations and QA teams.

Full Stack and Cyber Security

As a Cyber Security student, I find full stack development directly relevant to my field. Most modern cyber attacks target web applications — SQL injection targets the database layer, XSS (Cross-Site Scripting) targets the front end, broken authentication targets the back-end session management, and insecure APIs expose sensitive data. Understanding how each layer of a full stack application is built makes it much easier to identify where vulnerabilities exist and how to fix them. Many roles in Cyber Security — such as penetration tester or application security engineer — require a solid understanding of how web applications are developed.

🗄️ Database

Technologies: MySQL · PostgreSQL · MongoDB · SQLite

Stores and retrieves all application data permanently.

Common Full Stack Technology Combinations

In practice, developers often use a recognised combination of technologies that work well together. These combinations are sometimes given acronym names:

  • MERN Stack — MongoDB, Express.js, React, Node.js. A popular JavaScript-only stack where both the front end and back end use JavaScript.
  • MEAN Stack — MongoDB, Express.js, Angular, Node.js. Similar to MERN but uses Angular instead of React for the front end.
  • LAMP Stack — Linux, Apache, MySQL, PHP. A traditional and widely used stack, especially for content management systems like WordPress.
  • Django Stack — Python, Django, PostgreSQL. A clean and powerful stack favoured for data-heavy applications and rapid development.

For this module, the focus has been on the front-end layer — HTML, CSS, Bootstrap and JavaScript — which is the foundation needed before moving into back-end development and databases.
